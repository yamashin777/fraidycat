# Fraidycat Clone

YouTubeチャンネルやRSSフィードをまとめてチェックできるWebアプリです。オリジナルの[Fraidycat](https://fraidyc.at/)にインスパイアされ、Chrome拡張機能のサポート終了に伴い独自に開発しました。

単一のHTMLファイルで動作し、PC・iPhone・iPadのブラウザに対応しています。データはGitHubリポジトリに同期され、複数端末で同じフォロー一覧を共有できます。

## 特徴

- **チャンネル・RSS一括管理** — YouTube・ブログ・Podcastなど形式を問わず登録可能
- **GitHub同期** — フォロー一覧をGitHubに保存し、PC・スマホ・タブレット間で自動共有
- **新着順タイムライン** — 全チャンネルの動画を投稿日時でフラットに表示
- **動画時間の自動取得** — YouTube Data API v3を使って再生時間を取得
- **チャンネル登録者数表示** — アイコン下に登録者数を万単位で表示（月1回自動更新）
- **ライブ・ショート・配信予定の判別** — 色分けとカウントダウンで表示
- **更新頻度の指定** — チャンネルごとに5分〜6時間の取得間隔を設定
- **個別時刻指定** — 「毎日8:00と20:00に取得」のように時刻を指定して取得
- **タグ・メモ管理** — チャンネルにタグとメモを登録して整理
- **マーク色** — チャンネルを赤・青・緑・黄で色付けして強調
- **複数プロキシによる安定取得** — 4つのCORSプロキシを分散利用しレート制限を回避
- **JSONエクスポート／インポート** — 端末間でのデータ受け渡し・バックアップ
- **スマホ対応** — iPhone・iPadのSafariやBraveでも快適に動作
- **ダークモード対応**

## 使い方

### Webアプリとして使う（GitHub Pages）

1. `fraidycat.html` を `index.html` にリネームしてGitHubリポジトリにアップロード
2. Settings → Pages → Deploy from a branch（main）を設定
3. `https://ユーザー名.github.io/リポジトリ名` でアクセス

### ローカルで使う

`fraidycat.html` をブラウザで直接開くだけで動作します。GitHub同期を使わない場合もデータは端末内（localStorage）に保存されます。

## GitHub同期の設定

フォロー一覧を複数端末で共有するには、GitHub同期を設定します。

1. GitHubで同期用のリポジトリを用意する（このリポジトリでも可）
2. [Personal Access Token](https://github.com/settings/tokens) を作成する
   - `repo` 権限（リポジトリの読み書き）を付与
3. アプリのドロワー（☰）→「GitHub同期設定」を開く
4. リポジトリ名（例: `ユーザー名/fraidycat`）とトークンを入力
5. 以降、フォローの追加・編集が自動的に `data/follows.json` に保存されます

> **注意**: Personal Access Tokenは端末ごとにブラウザ内に保存されます。共有端末では設定しないでください。

## 主な機能

### フォローの管理

| 操作 | 方法 |
|------|------|
| 追加 | 「+ フォローを追加」→ URL・名前・タグ・メモ・更新頻度を入力 |
| 一括インポート | 「一括インポート」→ テキスト形式で複数登録 |
| 編集 | チャンネルをクリックして展開 → 更新頻度・個別指定・メモ等を編集 |
| 削除 | チャンネル右端の「✕」をクリック |
| マーク色 | 「☆」をクリックして赤・青・緑・黄から選択 |

### テキスト一括インポートの形式

```
#タグ名
https://www.youtube.com/feeds/videos.xml?channel_id=XXXX チャンネル名
https://www.youtube.com/feeds/videos.xml?channel_id=YYYY チャンネル名

#別のタグ
https://example.com/feed ブログ名
```

### 表示モード

| モード | 説明 |
|--------|------|
| 📋 チャンネル順 | 全チャンネルをカード表示（チャンネルごとに最新動画をまとめて表示） |
| 🕐 新着順 | 全動画を投稿日時順でタイムライン表示 |
| ⏬ 長い順 | 再生時間の長い動画から順に表示 |
| ⏫ 短い順 | 再生時間の短い動画から順に表示 |
| 🆕 登録新しい順 | 最近登録したチャンネルを上に表示 |
| 🕰 登録古い順 | 最初に登録したチャンネルを上に表示 |
| 👥 登録者数順 | チャンネル登録者数の多い順に表示 |

### 更新頻度

各チャンネルの取得間隔を設定できます。アプリのタブを開いている間、設定した間隔で自動的にRSSを取得します。

| ラベル | 取得間隔 |
|--------|---------|
| 5分 | 5分ごと |
| 15分 | 15分ごと |
| 30分 | 30分ごと |
| 1時間 | 1時間ごと |
| 6時間 | 6時間ごと |

### 個別時刻指定

更新頻度とは別に、チャンネルごとに「毎日決まった時刻に取得」を指定できます。チャンネルを展開し、「個別指定」欄に時刻を入力します。

```
8:00          → 毎日8時に取得
8:00,20:00    → 毎日8時と20時に取得
```

- 指定時刻を過ぎていてまだ取得していなければ、ページを開いた時点で取得します（取りこぼし防止）
- 設定したチャンネルにはチャンネル順表示で「個別指定」バッジが表示されます
- サイドバー／ドロワーの「個別指定」フィルターで絞り込めます

### YouTube Data API v3の設定（動画時間・アイコン・登録者数の取得に必要）

1. [Google Cloud Console](https://console.cloud.google.com/) でプロジェクトを作成
2. 「APIとサービス」→「ライブラリ」→ `YouTube Data API v3` を有効化
3. 「認証情報」→「APIキーを作成」→ キーをコピー
4. アプリのドロワー →「YouTube APIキー設定」からキーを登録

> **無料枠**: 1日1万リクエストまで無料。動画時間・アイコン・登録者数の取得に使用します。
> **登録者数**: 一度取得すると30日間は再取得せず、月1回のペースで自動更新されます。

### PC ↔ スマホのデータ共有

GitHub同期を設定していれば、各端末でページを開くだけで自動的に同期されます。同期を使わない場合は、JSONエクスポート／インポートで受け渡しできます。

1. PCでドロワー →「JSONエクスポート」→ ファイルをダウンロード（iOSでは画面に表示してコピー）
2. スマホに送信
3. スマホのブラウザで「インポート」→ JSONを選択

## 動画の色分け・バッジ

| 表示 | 意味 |
|------|------|
| 🔴 `Live` | ライブ配信中、またはタイトルに「LIVE」「生放送」を含む |
| 🔴 `予定 M/D HH:MM` | 配信予定（プレミア公開・ライブ予約）。予定時刻まではカウントダウン表示 |
| 🔵 `[ショート]` | YouTubeショート動画（`/shorts/` URL形式） |
| 再生時間（例: `14分45秒`） | 通常動画の再生時間 |

## RSS取得の仕組み

ブラウザの同一オリジンポリシー（CORS）により、YouTubeやブログのRSSを直接取得できないため、CORSプロキシを経由して取得します。本アプリは4つの公共プロキシ（allorigins.win / corsproxy.io / rss2json.com / feedproxy.google.com）を併用し、チャンネルごとに開始プロキシをずらして負荷を分散します。あるプロキシがレート制限（429）やエラーを返した場合は、一定時間そのプロキシをスキップして別のプロキシを使います。

> 公共プロキシは混雑時にレート制限がかかることがあります。より安定させたい場合は、Cloudflare Workersなどで自前のプロキシを用意してプロキシリストに追加する方法もあります。

## データの保存場所

| 保存先 | 内容 |
|--------|------|
| ブラウザのlocalStorage | フォロー一覧・設定（端末ごと） |
| GitHubリポジトリ `data/follows.json` | フォロー一覧（同期用・全端末共通） |

> **バックアップ推奨**: 重要なデータは定期的にJSONエクスポートしておくと、万一の同期トラブル時に復旧できます。GitHubのコミット履歴も過去状態の復元に利用できます。

## 技術仕様

- 単一HTMLファイル（HTML / CSS / JavaScript、外部フレームワーク不使用）
- データ保存: localStorage ＋ GitHub Contents API
- RSS取得: 複数CORSプロキシ経由、同時取得数を制限
- YouTube Data API v3: 動画時間・アイコン・ライブ状態・登録者数の取得
- 対応ブラウザ: PC（Chrome / Brave / Safari等）・iPhone / iPad（Safari / Brave）
